# Git 提交规范

**格式：** `[emoji] type: 简短描述`

【重要】允许分多次提交

## 示例

```
✨ feat: 实现用户登录功能
🐛 fix: 修复并发请求导致重复注册的问题
♻️ refactor: 重构数据库连接池管理，连接复用率提高 40%
📝 docs: 编写项目启动功能的文档
```

## 类型

| 类型 | Emoji | 说明 |
|------|-------|------|
| `feat` | ✨ | 新功能 |
| `fix` | 🐛 | 修复 Bug |
| `refactor` | ♻️ | 重构代码 |
| `docs` | 📝 | 文档更新 |
| `perf` | ⚡ | 性能优化 |
| `test` | ✅ | 测试相关 |
| `chore` | 🔧 | 构建/工具 |
| `style` | 🎨 | 代码格式 |
| `ci` | 👷 | CI/CD |
| `data` | 📚 | 数据/资源文件 |
| `milestone` | 🎯 | 里程碑 |
